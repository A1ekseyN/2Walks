Теоретически ~1000 невалидов подряд → RecursionError; на практике — загрязнённые стектрейсы при любой ошибке + неидиоматично. После: каждая функция-меню обёрнута в `while True` с `continue` на ошибке/невалиде и `return` на успехе. Pattern: рендер меню внутри loop'а (чтобы reprompt перерисовывал опции, а не показывал пустой `>>>`). Длинные меню получили helper-функции для рендеринга: `Adventure._render_adventure_menu(self)` (45 строк рендеринга вынесены), `gym._render_gym_menu(state, skill_options)`. Это держит тело `while` компактным. В gym.py попутно: (1) узкий `except ValueError` вместо широкого `except Exception` (последний маскировал любые баги); (2) удалён рекурсивный вызов `gym_menu(self._state)` из `Skill_Training.check_requirements` при нехватке ресурсов — внешний loop в gym_menu сам перерисует меню через `continue`, без двойного рендера. В shop.py 4 одинаковые stub-функции (clothes_head/jacket/pants/gloves) слиты в общий `_clothes_stub(label, header)` (DRY — уйдёт в 4.7 при полном рерайте Shop). В adventure.py устранён ping-pong рекурсии `adventure_menu` ↔ `adventure_choice`: `adventure_choice_confirmation` теперь возвращает bool (True если стартовало, False для "назад"), `adventure_choice` обрабатывает результат через `continue`/`return`.
