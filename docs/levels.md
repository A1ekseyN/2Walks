# Система уровней персонажа

Документ описывает, как в 2Walks реализована прогрессия уровней персонажа: где хранится таблица порогов, как рассчитывается уровень, что происходит при повышении, и как работает прогресс-бар.

---

## 1. Где это живёт

Вся логика уровней — в одном файле `level.py`, в классе `CharLevel`.

Поля в глобальном словаре `char_characteristic` (`characteristics.py:118-123`), связанные с уровнями:

| Поле | Назначение |
|---|---|
| `char_level` | Текущий уровень персонажа (целое число, начиная с 0) |
| `char_level_up_skills` | Пул не распределённых очков навыков, накопленных за lvl up |
| `lvl_up_skill_stamina` | Прокачанное очками значение навыка "Stamina" от уровня |
| `lvl_up_skill_energy_max` | Прокачанное очками значение навыка "Energy" от уровня |
| `lvl_up_skill_speed` | Прокачанное очками значение навыка "Speed" от уровня |
| `lvl_up_skill_luck` | Прокачанное очками значение навыка "Luck" от уровня |

Источник данных для расчёта — `char_characteristic['steps_total_used']` (общее количество "потраченных" шагов за всю историю игры).

---

## 2. Таблица порогов

Жёстко задана в классе `CharLevel` (`level.py:8-21`) как `LEVEL_THRESHOLDS`:

| Уровень | Нужно потрачено шагов (`steps_total_used`) | Разница с предыдущим |
|---|---:|---:|
| L1 | 10 000 | +10 000 |
| L2 | 20 000 | +10 000 |
| L3 | 50 000 | +30 000 |
| L4 | 100 000 | +50 000 |
| L5 | 250 000 | +150 000 |
| L6 | 500 000 | +250 000 |
| L7 | 750 000 | +250 000 |
| L8 | 1 000 000 | +250 000 |
| L9 | 1 300 000 | +300 000 |
| L10 | 1 650 000 | +350 000 |
| L11 | 2 050 000 | +400 000 |
| L12 | 2 500 000 | +450 000 |

Максимальный уровень — **12**. Кривая прогрессии построена на основе реального опыта регулярной ходьбы и оттестирована в ходе игры.

**Внутренний формат словаря:** ключи в `LEVEL_THRESHOLDS` идут от 0 до 11 и нумеруют уровни со сдвигом на 1. То есть ключ `0` — это порог для перехода на уровень 1, ключ `11` — на уровень 12. Это следствие алгоритма расчёта (см. раздел 3).

---

## 3. Алгоритм расчёта уровня

Функция `calculate_level_from_total_used_steps()` (`level.py:48-59`):

```python
def calculate_level_from_total_used_steps(self):
    used_steps = self.total_used_steps
    new_level = 0

    for level, threshold in sorted(self.LEVEL_THRESHOLDS.items()):
        if used_steps > threshold:
            new_level = level + 1
        else:
            break

    return new_level
```

**Логика:**
1. Начинаем с `new_level = 0`.
2. Перебираем пороги в порядке возрастания.
3. На каждой итерации: если потраченных шагов **строго больше** текущего порога, устанавливаем `new_level = индекс_порога + 1`.
4. Как только находим порог, который ещё не пройден — выходим из цикла.
5. Возвращаем накопленный `new_level`.

**Пример:**
- `steps_total_used = 75 000` → пройдены пороги 10 000 и 20 000 (индексы 0 и 1) и 50 000 (индекс 2), не пройден 100 000 (индекс 3) → `new_level = 2 + 1 = 3` → уровень 3.

## 4. Когда происходит перерасчёт уровня

`update_level()` (`level.py:61-69`) вызывается **автоматически** внутри `level_status_bar()` (`level.py:142-146`), который в свою очередь вызывается из `status_bar()` (`functions.py:66`).

Это значит: уровень пересчитывается **каждый раз**, когда игрок возвращается в главное меню и видит строку статуса — то есть практически после любого действия.

Если `new_level != self.level`:
1. Считается разница `level_up_difference = new_level - self.level` (может быть больше 1, если за одно действие прошли сразу несколько порогов).
2. Обновляется `char_characteristic['char_level']`.
3. `char_characteristic['char_level_up_skills'] += level_up_difference` — игрок получает очки на распределение.
4. Печатается сообщение: `Персонаж повысил свой уровень до N уровня.`

## 5. Очки навыков за lvl up

За каждый пройденный уровень начисляется **+1 очко** в пул `char_characteristic['char_level_up_skills']`.

Потратить очко можно через меню `menu_skill_point_allocation()` (`level.py:97`), которое вызывается командой `u` / `г` из главного меню игры (`game.py:109`).

Доступные навыки и эффект одного очка:

| # | Навык | Эффект за 1 очко |
|---|---|---|
| 1 | Stamina | + 1 % к общему количеству шагов за день |
| 2 | Energy | + 1 единица к максимальному запасу энергии (`energy_max`) |
| 3 | Speed | + 1 % к скорости выполнения всех активностей (работа, тренировки, приключения) |
| 4 | Luck | + 1 % к удаче (влияет на шанс выпадения предметов и их качество) |

Распределение точечное: игрок выбирает один навык из списка, очко списывается из `char_level_up_skills`, соответствующее поле `lvl_up_skill_*` увеличивается на 1. Если очков нет (`char_level_up_skills == 0`), меню показывает текущие значения навыков, но не даёт прокачать.

После выхода из меню `0` (Назад) распределение можно продолжить позже — очки не сгорают.

## 6. Прогресс-бар

Функция `progress_to_next_level()` (`level.py:71-81`) возвращает процент прогресса до следующего уровня:

```python
current_threshold = 0 if self.level == 0 else self.LEVEL_THRESHOLDS[self.level - 1]
next_threshold = self.LEVEL_THRESHOLDS.get(self.level, float('inf'))

steps_into_level = self.total_used_steps - current_threshold
steps_needed = next_threshold - current_threshold

progress_percentage = (steps_into_level / steps_needed) * 100 if steps_needed > 0 else 100
```

**Как читать:**
- `current_threshold` — порог текущего уровня (на L0 это 0, на L1 — значение `LEVEL_THRESHOLDS[0] = 10 000`, и т.д.).
- `next_threshold` — порог следующего уровня.
- `steps_into_level` — сколько шагов потрачено сверх порога текущего уровня.
- `steps_needed` — расстояние между текущим и следующим порогом.
- Процент — отношение одного к другому.

Функция `progress_bar()` (`level.py:83-88`) рисует визуальную шкалу из `#` и `-` длиной 33 символа:

```
Уровень: 3 [##########-----------------------] 30.00 % [+ 2 Skill points]
```

Надпись `[+ N Skill points]` появляется справа, если в пуле есть не распределённые очки (`progress_bar_lvl_up_message()`, `level.py:90-95`).

## 7. Что сохраняется в сейв

В CSV/JSON/Google Sheets попадают 6 полей:

- `char_level`
- `char_level_up_skills`
- `lvl_up_skill_stamina`
- `lvl_up_skill_energy_max`
- `lvl_up_skill_speed`
- `lvl_up_skill_luck`

Сама таблица `LEVEL_THRESHOLDS` в сейв не идёт — она захардкожена в коде и одинакова для всех персонажей.

## 8. Файлы, относящиеся к системе уровней

- `level.py` — класс `CharLevel`, таблица порогов, меню распределения очков.
- `characteristics.py:118-123` — поля `char_level*` и `lvl_up_skill_*` в `char_characteristic`.
- `functions.py:66` — точка вызова `level_status_bar()` из `status_bar()`.
- `bonus.py:22` — `level_steps_bonus()` использует `lvl_up_skill_stamina` для расчёта бонуса к шагам.
- `game.py:109` — команда `u` / `г` главного меню, открывающая `menu_skill_point_allocation`.
