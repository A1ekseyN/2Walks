# Доступ извне через Tailscale

Справка по доступу к домашнему серверу и маршрутизации трафика **из любой точки интернета** через [Tailscale](https://tailscale.com) — VPN-оверлей поверх WireGuard.

## Контекст

Домашний Ubuntu-сервер (`aleksey@192.168.0.155`, hostname `aleksey-H61M-DS2H`) держит web-игру 24/7 (systemd `2walks.service`, порт `8008` — см. [`local_setup.md`](local_setup.md) раздел «Production»). Дома он доступен по Wi-Fi, но снаружи — нет. Плюс есть рабочий сайт с **whitelist по IP**: он пускает только с домашнего статического IP, поэтому из другого места (кафе, мобильный интернет) — отказ.

**Tailscale закрывает обе задачи одной установкой** на сервере:

| Часть | Что даёт | Механизм |
|---|---|---|
| **Часть 1** | Весь трафик устройства выходит в интернет через домашний IP (→ рабочий сайт пускает) | сервер как **exit node** |
| **Часть 2** | Доступ к web-игре с телефона из любой точки | сервер как **хост в tailnet** |

> ⚠️ **Почему именно VPN, а не проброс порта в интернет:** у web-игры **нет авторизации** (задача 4.55 не закрыта). Любой, кто достучится до `:8008`, может менять сейв. Tailscale не создаёт публичной поверхности атаки — доступ только внутри твоей приватной tailnet. Голый проброс порта 8008 наружу — **нельзя**.

> 📌 **Статический IP** нужен только для Части 1 (рабочий сайт проверяет именно его). Для Части 2 (игра) static IP не требуется — Tailscale строит оверлей поверх любого подключения.

> SSH к серверу интерактивный — все серверные команды запускать вручную (например `! ssh aleksey@192.168.0.155`), не из автоматизации.

> **Текущий статус (2026-06-25):**
> - **Часть 1 (exit node для рабочих задач)** — ✅ **настроено и проверено.** `tailscale status` показывает `aleksey-h61m-ds2h … active; exit node`, Tailscale-IP сервера — `100.71.10.117`. Проверка с клиента (раздел 1.5) выполнена: телефон с включённым exit node показывает домашний статический IP **`194.156.248.138`** (до активации был IP моби-оператора). Трафик идёт через дом — рабочий сайт с whitelist пустит.
> - **Часть 2 (iPhone + доступ к web-игре)** — **ещё не настраивалась.** MagicDNS / HTTPS / `tailscale serve` не включались; iPhone в tailnet пока не добавлен. Сейчас в tailnet только сервер и Mac (`macbook-pro---macbook`, `100.105.39.108`).

## Часть 1 — Маршрутизация всего трафика (exit node)

**Цель:** заходить на рабочий сайт с whitelist по IP из любого места. Устройство гонит весь интернет-трафик через домашний сервер, и сайт видит **домашний статический IP** вместо IP текущей локации.

**Предусловие:** в whitelist рабочего сайта прописан именно **домашний** статический IP (тот, где стоит сервер). Если разрешён офисный/другой IP — домашний exit node не поможет, узел должен стоять в той сети.

### 1.1. На сервере: включить IP-форвардинг

Без этого exit node молча не работает (трафик не пересылается).

```bash
echo 'net.ipv4.ip_forward = 1' | sudo tee -a /etc/sysctl.d/99-tailscale.conf
echo 'net.ipv6.conf.all.forwarding = 1' | sudo tee -a /etc/sysctl.d/99-tailscale.conf
sudo sysctl -p /etc/sysctl.d/99-tailscale.conf
```

### 1.2. На сервере: установить Tailscale и поднять exit node

```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --advertise-exit-node
```

`tailscale up` выведет URL — войти аккаунтом (Google / почта / GitHub). Демон `tailscaled` ставится как systemd-сервис, переживает перезагрузку.

### 1.3. В админке (login.tailscale.com) — два обязательных действия

Без них exit node **не заработает**:

1. **Approve exit node.** Machines → выбрать сервер → ⋯ → **Edit route settings** → одобрить Exit node. Пока не одобришь — клиенты его не видят.
2. **Disable key expiry.** Тот же сервер → **Disable key expiry**. Это узел 24/7: иначе через ~180 дней ключ протухнет, машина выпадет из tailnet и пропадут **и exit node, и игра** одновременно.

### 1.4. На клиенте (iPhone / ноутбук)

1. Установить приложение **Tailscale**, войти **тем же аккаунтом**.
2. Включить выход через дом: в приложении → **Exit Node → выбрать домашний сервер**.
3. Выключать exit node, когда не нужен (тогда трафик идёт напрямую — быстрее).

### 1.5. Проверка

С устройства при **включённом** exit node открыть любой «what is my IP» — должен показаться **домашний статический IP** (публичный IP домашнего интернет-канала, где стоит сервер — **не** внутренний `192.168.0.155` и **не** Tailscale-адрес `100.71.10.117`). Это ровно тот IP, что в whitelist рабочего сайта. Значит сайт пустит. Бонус: Tailscale при exit node гонит через туннель и DNS — утечки реального IP/DNS мимо туннеля нет.

> ✅ **Проверено 2026-06-25.** Телефон с включённым exit node показал **`194.156.248.138`** — домашний статический IP (до активации был IP моби-оператора). Этот же IP виден в `tailscale status` как direct-endpoint сервера (`194.156.248.138:NNNNN`) и как egress-IP домашнего Mac — оба за одним роутером. Часть 1 реально работает.

### 1.6. Управление вкл/выкл exit node

**Ключевая идея — две стороны, не путать:**

- **Сервер «раздаёт»** себя как exit node (`--advertise-exit-node`) — постоянное свойство, «доступен как выход».
- **Клиент решает, пользоваться им или нет** — это и есть повседневный тумблер «включить/выключить маршрутизацию».

То есть обычное вкл/выкл — **действие на клиенте**, оно встроено в Tailscale, своего кода/веб-интерфейса не требует.

**Повседневный тумблер (на клиенте):**

```bash
# Ноут (Mac / Linux): включить / выключить выход через дом
tailscale set --exit-node=aleksey-h61m-ds2h     # включить (имя машины; или IP 100.71.10.117)
tailscale set --exit-node=                       # выключить (пустое значение)

tailscale status                                 # проверить текущий exit node
```

На **iPhone** — приложение Tailscale → **Exit Node → выбрать сервер / None**. Один тап, без консоли.

**Серверная сторона (раздавать или нет) — при активном Tailscale меняется одной командой, без full re-up:**

```bash
sudo tailscale set --advertise-exit-node=false   # перестать быть exit node
sudo tailscale set --advertise-exit-node=true    # снова стать exit node
```

Удалённо — через `ssh aleksey@192.168.0.155 "sudo tailscale set --advertise-exit-node=false"`.

> **Нужен ли вкл/выкл для безопасности? — Нет.** Держать exit node постоянно «раздающимся» — низкий риск: воспользоваться им могут **только устройства в твоей tailnet** (твой аккаунт), снаружи никто не подключится — сама tailnet и есть граница безопасности. Тумблер нужен **для скорости/удобства**, а не для защиты: при включённом exit node весь трафик клиента идёт через домашний канал и работает медленнее, поэтому держат выключенным и включают только когда нужен рабочий сайт.

> **Свой веб-UI/API для тумблера — скорее НЕ нужен:** нативный тумблер уже есть (iPhone-приложение / `tailscale set`). Если всё же хочется «кнопку», помни: эндпоинт, дёргающий `sudo tailscale set ...`, — это веб-ручка с привилегированными командами. Делать только hardened: бинд **только на tailnet-интерфейс** (не `0.0.0.0`), **авторизация**, **узкое правило sudoers** (ровно `tailscale set --advertise-exit-node=*`, не полный sudo), отдельным микросервисом — **не** подмешивая в `2walks.service` (у игры auth нет — задача 4.55).

### Грабли и нюансы

- **Скорость.** При включённом exit node весь интернет устройства идёт через домашний аплоад-канал → пинг/скорость упираются в исходящую скорость домашнего интернета. Для рабочего сайта/браузинга норм, для тяжёлых загрузок медленно — потому и удобен тумблер on/off.
- **SSH к серверу не сломается** — сервер *раздаёт* exit node, но сам через него не ходит.
- **Политика работодателя.** Рабочий трафик идёт через личный сервер — формально может противоречить правилам компании. Раз доступ по IP выдан официально — скорее всего ок, но имей в виду.
- **Split-tunnel (только рабочий сайт через дом, остальное напрямую)** — отдельная задача: либо WireGuard с `AllowedIPs = <IP_сайта>/32`, либо Tailscale app-connector. Здесь пока «весь трафик»; split добавим при необходимости.

## Часть 2 — Доступ к web-игре через Tailscale

> ⏳ **Ещё не настроено (на 2026-06-25).** Сделана только Часть 1 (exit node для рабочих задач). iPhone в tailnet не добавлен, MagicDNS / HTTPS / `tailscale serve` не включались. Раздел ниже — план на будущее. Tailscale-IP сервера уже известен (`100.71.10.117`), так что игра технически достижима из tailnet по `http://100.71.10.117:8008/` сразу после добавления клиента — отдельный re-up на сервере не нужен.

**Цель:** открывать игру с iPhone из любой точки, без публичного доступа и без проброса портов.

**Менять код не нужно:** `2walks.service` уже слушает `0.0.0.0:8008`, поэтому игра автоматически доступна и на Tailscale-интерфейсе сервера.

### 2.1. На сервере: Tailscale

Если уже поставил по **Части 1** — пропусти, база общая. Если нужен **только** доступ к игре (без exit node):

```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
```

Узнать адрес сервера в tailnet:

```bash
tailscale ip -4        # → 100.71.10.117 (текущий адрес сервера)
```

> Совет: в админке **Disable key expiry** для сервера (как в 1.3) — узел 24/7, ключ не должен протухать.

### 2.2. На iPhone

Установить приложение **Tailscale**, войти **тем же аккаунтом**. Телефон попадает в tailnet.

### 2.3. Открыть игру

Из любой точки (даже с мобильного интернета):

```
http://100.71.10.117:8008/
```

Если включить **MagicDNS** (админка → DNS → Enable MagicDNS) — по имени машины:

```
http://aleksey-h61m-ds2h:8008/
```

### 2.4. Опционально — HTTPS без публичного доступа

Чтобы Safari не ругался на `http://` и был валидный TLS-сертификат:

1. Админка → DNS → включить **HTTPS Certificates** (и MagicDNS).
2. На сервере:
   ```bash
   sudo tailscale serve --bg 8008
   ```

Игра откроется по `https://<имя-машины>.ts.net` — настоящий HTTPS, но всё ещё **только внутри tailnet**, наружу ничего не торчит.

### Нюансы

- **Игра доступна независимо от exit node** (Часть 1). Сервер — пир tailnet сам по себе; включён exit node или нет, игра достижима по своему tailscale-адресу.
- **Домашний Wi-Fi-доступ** (`192.168.0.155:8008` / `aleksey-H61M-DS2H.local:8008`) продолжает работать как раньше — Tailscale его не заменяет, а дополняет для доступа извне.
- **Нет auth** — именно поэтому игра живёт только в приватной tailnet. Публичный URL появится не раньше, чем будет закрыта задача 4.55 (авторизация).

## Резюме команд

```bash
# === Сервер (один раз) ===
# IP-форвардинг (нужно для exit node, Часть 1)
echo 'net.ipv4.ip_forward = 1' | sudo tee -a /etc/sysctl.d/99-tailscale.conf
echo 'net.ipv6.conf.all.forwarding = 1' | sudo tee -a /etc/sysctl.d/99-tailscale.conf
sudo sysctl -p /etc/sysctl.d/99-tailscale.conf

# Установка + поднятие (exit node + игра одной командой)
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --advertise-exit-node

tailscale ip -4                 # адрес сервера в tailnet (для игры) → 100.71.10.117
sudo tailscale serve --bg 8008  # опционально: HTTPS для игры

# === Админка login.tailscale.com (один раз) ===
# - Approve exit node (Edit route settings)
# - Disable key expiry для сервера
# - Enable MagicDNS + HTTPS Certificates (опционально)

# === iPhone ===
# - Приложение Tailscale, вход тем же аккаунтом
# - Exit Node → домашний сервер (для рабочего сайта)
# - Игра: http://100.71.10.117:8008/  или  https://<host>.ts.net
```
